= Yousign eSignature connectors
:description: Bonita connectors for the Yousign eSignature API v3. Create, manage, track, and download electronic signature requests directly from your Bonita processes.

{description}

The Bonita Yousign Connectors are available for **Bonita 10.2 Community (2024.3)** version and above.

== Overview

The Yousign connector provides 8 operations covering the complete electronic signature lifecycle:

* **Create From Template** -- Creates a signature request from a pre-configured Yousign template
* **Activate** -- Activates a signature request, sending email invitations to all signers
* **Get Status** -- Retrieves the current status of a signature request and its signers
* **Cancel** -- Cancels an active signature request with an optional reason
* **Download Document** -- Downloads a single signed document or all documents as a ZIP archive
* **Download Audit Trail** -- Downloads the legally binding audit trail PDF for a completed request
* **List** -- Lists signature requests with filters (status, external ID) and cursor-based pagination
* **Register Webhook** -- Registers a webhook endpoint to receive real-time Yousign event notifications

=== Yousign signature request lifecycle

A signature request follows this lifecycle:

----
draft --> approval --> ongoing --> done
                         |
                         +--> declined
                         +--> canceled
                         +--> expired
----

* **draft** -- The request has been created but not yet activated
* **approval** -- The request is pending internal approval (if configured)
* **ongoing** -- Signers have been invited and the request is awaiting signatures
* **done** -- All signers have signed; documents are available for download
* **declined** -- A signer has declined to sign
* **canceled** -- The request was canceled by the initiator
* **expired** -- The request passed its expiration date without completion

=== Recommended process pattern

The typical Bonita process flow for electronic signatures follows this pattern:

----
[Start] --> [Create From Template] --> [Activate] --> [Polling Loop / Webhook] --> [Get Status]
                                                                                       |
                                                          +-----(done)------> [Download Document] --> [Download Audit Trail] --> [End]
                                                          +----(declined)---> [Handle Decline] --> [End]
                                                          +----(canceled)---> [Handle Cancel] --> [End]
                                                          +----(expired)----> [Handle Expiry] --> [End]
----

You can monitor progress using either:

1. **Polling pattern** -- Use a timer-based loop with Get Status to check periodically
2. **Webhook pattern** -- Register a webhook to receive instant notifications via a Bonita REST API extension

== Getting started

Add the connector as an extension dependency to your Bonita project. Import the `.jar` file via *Import from file* in Bonita Studio.

=== Import into Bonita Studio

1. Build the connector: `./mvnw install -DskipTests`
2. In Bonita Studio, go to **Development > Connectors > Import connector...**
3. Select the JAR file: `target/bonita-connector-yousign-1.0.0-beta.1.jar`
4. The 8 Yousign operations will appear under the **Yousign** category

=== Maven coordinates

[source,xml]
----
<dependency>
    <groupId>org.bonitasoft.connectors</groupId>
    <artifactId>bonita-connector-yousign</artifactId>
    <version>1.0.0-beta.1</version>
</dependency>
----

=== Authentication setup

Yousign uses **Bearer token authentication**. Each API call includes the API key in the `Authorization: Bearer <apiKey>` header.

To obtain an API key:

1. Sign up at https://yousign.com/
2. Go to **Settings > API Keys** in the Yousign dashboard
3. Create an API key for your environment (sandbox or production)

IMPORTANT: The API key is treated as a secret. It is configured via a **Password** widget in Bonita Studio, which masks the value in the UI.

== Create From Template (`yousign-create-from-template`)

Creates a new signature request based on a pre-configured Yousign template. The template defines the document(s), signature fields, and placeholders. This operation maps signers to template roles and fills template text fields.

This is typically the first step in a signature workflow.

== Error handling

All operations set `success=false` and populate `errorMessage` on failure. The connector never throws exceptions to the Bonita engine -- errors are always captured in the output parameters.

[caption=HTTP error handling]
|===
|HTTP Code |Behavior

|200/201
|Success -- parse response and populate outputs

|400
|Fail immediately -- bad request (validation error in your input parameters)

|401
|Fail immediately -- authentication error (invalid or expired API key)

|403
|Fail immediately -- access denied (insufficient permissions)

|404
|Fail immediately -- resource not found (invalid UUID)

|429
|Retry up to 3 times with exponential backoff -- rate limit exceeded

|500/502/503
|Retry up to 3 times with exponential backoff -- server error
|===

The retry policy uses exponential backoff with jitter: base delays of 1s, 2s, 4s plus random jitter to prevent thundering herd.

== Groovy code examples

=== Polling loop for signature completion

Use this pattern in a Bonita script task or connector output to poll until the signature request reaches a terminal state.

[source,groovy]
----
// In a Bonita loop connector (timer: 30 seconds interval)
// Check if the status is terminal to exit the loop
def isTerminal = isTerminal // output from Get Status connector
def status = status         // output from Get Status connector

// Loop exit condition (set in the loop configuration)
return isTerminal == true
----

=== Download and archive signed documents

Use this in a Groovy script task after the signature is completed.

[source,groovy]
----
// After Download Document connector, decode Base64 content to a Bonita document
import java.util.Base64

def documentContent = documentContent  // Base64 output from Download Document
def documentName = documentName        // filename output

// Decode and store as Bonita document
byte[] decodedBytes = Base64.getDecoder().decode(documentContent)

// Use Bonita API to attach as a document to the process
apiAccessor.getProcessAPI().attachDocument(
    processInstanceId,
    documentName,
    documentName,
    contentType,      // e.g., "application/pdf"
    decodedBytes
)
----

=== Webhook payload verification

If you set a webhook secret, verify incoming payloads in your Bonita REST API extension.

[source,groovy]
----
import javax.crypto.Mac
import javax.crypto.spec.SecretKeySpec

def verifyWebhookSignature(String payload, String signature, String secret) {
    def mac = Mac.getInstance("HmacSHA256")
    mac.init(new SecretKeySpec(secret.getBytes("UTF-8"), "HmacSHA256"))
    def computed = mac.doFinal(payload.getBytes("UTF-8")).encodeHex().toString()
    return computed == signature
}
----

== Process design patterns

=== Pattern 1: Simple signature with polling

----
[Start] --> [Create From Template] --> [Activate] --> [Timer Loop]
                                                          |
                                                    [Get Status] ---(not terminal)---> [Wait 30s] --> [Get Status]
                                                          |
                                                      (terminal)
                                                          |
                                           +--- done --> [Download Document] --> [Download Audit Trail] --> [End]
                                           +--- declined --> [Notify Initiator: Declined] --> [End]
                                           +--- expired --> [Notify Initiator: Expired] --> [End]
----

=== Pattern 2: Webhook-based notification

----
[Start] --> [Register Webhook] --> [Create From Template] --> [Activate] --> [Wait for Signal]
                                                                                   |
                                                                           (webhook event)
                                                                                   |
                                                                            [Get Status] --> [Gateway]
                                                                                                |
                                                                    +--- done --> [Download] --> [Archive] --> [End]
                                                                    +--- other --> [Handle] --> [End]
----

=== Pattern 3: Multi-signer sequential signing

----
[Start] --> [Create From Template (orderedSigners=true)] --> [Activate]
                                                                |
                                                          [Poll/Webhook Loop]
                                                                |
                                                          [Get Status]
                                                                |
                                                    (check signerStatusesJson)
                                                                |
                                                +--- all done --> [Download + Archive]
                                                +--- partial --> [Continue waiting]
                                                +--- declined --> [Cancel + Notify]
----

== Troubleshooting

=== Common errors

[cols="1,2,2"]
|===
|Error |Cause |Solution

|`Authentication error: invalid API key`
|The API key is invalid, expired, or for the wrong environment
|Verify the key in the Yousign dashboard. Ensure sandbox keys are used with the sandbox URL and production keys with the production URL.

|`Resource not found`
|The signatureRequestId or documentId UUID does not exist
|Verify the UUID is correct. Check that the resource has not been deleted.

|`Bad request: invalid template`
|The template ID is invalid or the signer label does not match any role in the template
|Verify the template exists in your Yousign account. Check that the signerLabel exactly matches a role defined in the template.

|`Rate limit exceeded`
|Too many API calls in a short period
|The connector automatically retries 3 times with exponential backoff. If the error persists, reduce polling frequency or increase the timer interval.

|`Access denied`
|The API key does not have sufficient permissions
|Check the API key permissions in the Yousign dashboard. Ensure it has access to the required operations.
|===

=== Status transition errors

[cols="1,2"]
|===
|Scenario |Explanation

|Cannot activate a request
|The request must be in `draft` or `approval` status. It may have already been activated, canceled, or expired.

|Cannot cancel a request
|The request must be in `draft`, `approval`, or `ongoing` status. Terminal states (`done`, `declined`, `expired`) cannot be canceled.

|Cannot download documents
|Documents are only available when the request is in `done` status. Check the status before attempting to download.

|Webhook not receiving events
|Verify the endpoint URL is publicly accessible over HTTPS. Check the webhook status in the Yousign dashboard.
|===

== FAQ

*Can I use multiple templates in a single process?*

Yes. Each Create From Template operation is independent. You can create multiple signature requests in the same process, each using a different template.

*How do I handle multiple signers?*

Use the `additionalSignersJson` input to add extra signers beyond the primary signer. Each additional signer needs a JSON object with `info` (first_name, last_name, email) and a `label` matching a template role.

*What happens when a signer declines?*

The signature request status transitions to `declined`. The Get Status operation will return `isTerminal=true` and `status=declined`. No documents are signed.

*How long do signature requests stay valid?*

By default, Yousign signature requests expire after 6 months. You can set a custom expiration date using the `expirationDate` parameter in Create From Template.

*Can I use the connector in Bonita Community edition?*

Yes. The connector is compatible with Bonita 10.2 Community (2024.3) and above. All operations work with both Community and Enterprise editions.

*How do I switch from sandbox to production?*

Change the `baseUrl` parameter from `https://api-sandbox.yousign.app/v3` to `https://api.yousign.app/v3` and use a production API key. No other changes are needed.

*What is the maximum document size?*

Yousign supports documents up to 20 MB. For larger documents, consider splitting them or compressing before upload.

*How are documents returned?*

Documents are returned as Base64-encoded strings. Use `java.util.Base64.getDecoder().decode()` in a Groovy script to convert them back to binary for storage as Bonita documents.
